PROJECT: Pigeons

Overview

Pigeons is a desktop level application created for delivery companies to better assign and organize deliveries for the delivery people.

Mailer feature

We use an abstract Mailer class which provide a method that sends an email to a list of Person and another which sends an itinerary to the driver.
This class uses the JavaMail API to send emails from the address : pigeonscs2103@gmail.com.
We also created a new EmailCommand which directly uses the Mailer class in its redefinition of super.execute(). The information related to the itinerary are collected by the EmailCommand which creates an instance of RouteOptimization.

Autocompletion feature

We use an Autocompleter class which stores the list of Command and all the different fields (Name, Address, etc) created in the LogicManager. The Autocompleter can complete:

  • a command word, if it’s the first of a query

  • an option, starting by "n/" for example

  • a name, if the current query doesn’t match the first two cases: if we only have "John Doe" in the AddressBook we can complete the query by either "John" or "Doe" (this is useful to complete the FindCommand).

After having chosen between one this three cases, we generate a set of all possible suffixes that can be append to the current query. And from this set, we extract the longest common prefix which is the returned String of Autocompleter.autocomplete().
For example : if we want to complete the query find J and we have two persons in the address book called John and Jonathan, the set of possible suffixes will be {ohn, onathan} and the longest common prefix is o. So the completed query will be find Jo.

This class is used in CommandBox which sends the content of the command line and then append the result of the autocomplete operation.

Use case: Deliver a package to a specific customer

MSS

  1. User enters the package reference

  2. Address book outputs the customer’s information (availabilities, address)

  3. User clicks on the customer’s address

  4. Address book open the itinerary to the customer’s address on Google Maps

  5. User can drive to the customer’s location

    Use case ends.

Extensions

  • 2a. The given package reference is invalid.

    • 2a1. AddressBook shows an error message.

      Use case resumes at step 1.

Use case: Mapping the route for the day

MSS

  1. User requests to list the database ordered by date of delivery

  2. Address book outputs the list of deliveries

  3. User selects all the addresses and runs the “map” command on the selected addresses

  4. Address book displays a mapped out route for the day with an ideal time to begin the route

  5. User can start the delivery

    Use case ends.
